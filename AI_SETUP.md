# 🆓 Руководство по настройке AI

Игра поддерживает **2 AI-провайдера** с автоматическим переключением.

## Рекомендуется: Google Gemini (Основной)

**Почему Gemini?**
- ✅ Высокое качество генерации
- ✅ Отличная поддержка русского языка
- ✅ Быстрые ответы
- ✅ Модель: gemini-3-flash-preview

**Настройка (5 минут):**

1. Перейдите на https://aistudio.google.com/apikey
2. Войдите с Google аккаунтом
3. Нажмите "Create API Key"
4. Скопируйте ключ
5. Добавьте в файл `.env`:
   ```
   GEMINI_API_KEY=ваш_ключ_здесь
   ```

**Готово!** Перезапустите сервер и AI заработает.

---

## Альтернатива: Groq (Резервный)

**Почему Groq?**
- ✅ Очень быстрые ответы (< 1 секунды)
- ✅ Высокие лимиты (30 запросов/минуту)
- ✅ Не требуется кредитная карта
- ✅ Использует Llama 3.3 70B

**Настройка (2 минуты):**

1. Перейдите на https://console.groq.com/keys
2. Зарегистрируйтесь через Google/GitHub (бесплатно)
3. Нажмите "Create API Key"
4. Скопируйте ключ
5. Добавьте в файл `.env`:
   ```
   GROQ_API_KEY=gsk_ваш_ключ_здесь
   ```

**Готово!** Перезапустите сервер и AI заработает.

---

## 🔄 Как это работает

Приложение пробует провайдеры в таком порядке:

1. **Gemini** (если ключ есть) → Высокое качество, основной
2. **Groq** (если ключ есть) → Быстро, резервный
3. **Fallback** (всегда работает) → Предустановленный контент

Вам нужен только **ОДИН** провайдер! Приложение автоматически использует доступный.

---

## 📝 Ваш .env файл

Скопируйте `.env.example` в `.env` и добавьте хотя бы один ключ:

```bash
# Добавьте хотя бы один (или оставьте пустым для fallback режима)

GEMINI_API_KEY=ваш_ключ_gemini
GROQ_API_KEY=
```

---

## ✅ Тестирование

После добавления ключа:

1. Перезапустите сервер: `uvicorn main:app --reload`
2. Проверьте вывод в консоли:
   ```
   ✓ Gemini API configured
   ```
3. Начните игру - вы должны увидеть:
   ```
   [AI] Generated 9 events
   ```

Если видите "using fallback" - это тоже нормально! Игра работает без AI.

## 🆘 Решение проблем

**"No AI providers configured"**
- Добавьте хотя бы один API ключ в `.env`
- Убедитесь что файл `.env` существует (скопируйте из `.env.example`)

**"API error"**
- Проверьте правильность API ключа
- Убедитесь что скопировали полный ключ
- Попробуйте другой провайдер
